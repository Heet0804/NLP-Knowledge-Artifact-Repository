1. Stemming vs. Lemmatization

CriteriaStemmingLemmatization
Working MechanismRemoves prefixes or suffixes using simple rules.Uses vocabulary and morphological analysis to find the root word.
Computational ComplexityLowHigh (requires dictionary lookup)
StrengthsFast and easy to implementProduces meaningful and grammatically correct words
WeaknessesMay generate invalid root wordsSlower and requires language resources
Suitable ApplicationsSearch engines, indexing, autocomplete preprocessingPredictive text, chatbots, machine translation
Heet Lakhani (124BTCM1105)

2. One-Hot Encoding vs. Word Embeddings

CriteriaOne-Hot EncodingWord Embeddings
Working MechanismRepresents each word as a binary vector with one active value.Represents words as dense vectors learned from surrounding context.
Computational ComplexityHigh memory usage for large vocabulariesEfficient after training using fixed-size vectors
StrengthsSimple and easy to implementCaptures semantic similarity and contextual meaning
WeaknessesCannot represent relationships between wordsRequires training data and computational resources
Suitable ApplicationsBasic text classification, simple NLP tasksAutocomplete, predictive text, recommendation systems
Anand Raghvani (124BTCM1113)

3. TF vs. TF-IDF

CriteriaTFTF-IDF
Working MechanismCounts the number of times a word appears in a document.Adjusts word frequency using inverse document frequency.
Computational ComplexityO(n)O(n log n)
StrengthsSimple and computationally efficientHighlights important words by reducing common word influence
WeaknessesOveremphasizes frequently occurring common wordsRequires statistics from the complete corpus
Suitable ApplicationsBasic text retrieval and document analysisSearch engines, keyword extraction, document ranking
Anand Raghvani (124BTCM1113)

4. Word2Vec vs. FastText

CriteriaWord2VecFastText
Working MechanismLearns embeddings for complete words using CBOW or Skip-Gram.Learns embeddings using character-level n-grams along with words.
Computational ComplexityModerateHigh (due to character n-grams)
StrengthsFast training with good semantic representationHandles unknown words and morphologically rich languages like Gujarati
WeaknessesCannot generate vectors for unseen wordsRequires more memory and training time
Suitable ApplicationsDocument classification, recommendation systemsGujarati autocomplete, predictive text, spell correction, machine translation
